Provide a written summary of debugging suggestions and tips using Chrome when working with JavaScript.
There should be a minimum of 200 words and details provided.

Google's Chrome browser offers some great developer tools and I learned a lot from using them this chapter.

The one thing that I've dreaded using as a developer (in training) myself is the breakpoint tool. I didn't know how to use it at first and even though a tutorial was a click away, Chrome's developer tools was very user friendly and I picked it up quickly. The tool features a "step through and over" technique to help step through lines of code very easily and it even shows the input for a given variable or statement so you can see real-time / real-world events happening as it goes.

Another cool feature of Chrome's developer tools is the Sources page. The Sources page can show what files are being used in the webpage and even the code that is a given file. This is helpful because when there is an error in the code, a link through the error message can lead to the exact location of the error. This is more effective and efficient than having to traverse through a program, especially if that program contains a large amount of code.

Overall, I feel that Chrome's developer tools are a great asset in debugging and testing program.
